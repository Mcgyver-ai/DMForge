# DMForge — Impeccable Design Context

## Design Context

### Users

Online coaches and course creators (and agencies serving them) who sell via DMs on Instagram, WhatsApp, Messenger, and SMS. They are non-technical, busy, and revenue-focused: the job to be done is "get an AI appointment setter live in minutes and trust it before connecting my real account." They arrive skeptical of AI tools that could embarrass them in front of leads — the live-test-before-you-trust flow is the core confidence builder. Agency users additionally manage many client agents and need density and scanability.

### Brand Personality

**Confident, energetic, trustworthy.** The interface should feel like a high-momentum sales tool — speed, closing calls, pipeline moving — but with Stripe-grade polish and editorial restraint underneath, so the energy reads as competence rather than hype. Emotional goals: confidence ("this won't embarrass me"), momentum ("I shipped an agent in 60 seconds"), control ("I tested it live first").

### Aesthetic Direction

**Evolve the existing identity — don't replace it, and don't keep its AI-slop tells.**

Keep:

- Dark theme as the primary identity: deep navy base `#0B0B1A`, surfaces `#161630` / `#1F1F42`, borders `#2A2A55`
- Coral primary `#FF4D6D` as the signature brand color, purple `#6B5BFF` as a secondary accent
- Fraunces (display) + IBM Plex Sans (body) pairing — loaded via next/font (swapped from the original Space Grotesk + Inter during the anti-slop pass)

Remove / avoid (current AI-slop fingerprints to strip during redesign work):

- Gradient text (`.grad-text` coral→orange→purple) — replace with solid coral or typographic emphasis
- Neon glow shadows (`.glow-coral`, `.glow-purple`) — replace with intentional elevation (subtle tinted shadows, borders)
- Decorative radial gradient backgrounds used as filler
- Evenly-distributed accent colors; commit to coral-dominant with purple used sparingly and purposefully

Reference: **Stripe** — trustworthy, editorial, meticulous detail; density and hierarchy over decoration. Anti-reference: generic dark-mode AI SaaS (cyan/purple glows, gradient heroes, identical card grids).

Theme: dark-only for now (hardcoded `dark` class); light mode is a possible future evolution, not a current requirement.

### Design Principles

1. **Energy through motion and rhythm, not glow.** Convey momentum with purposeful micro-interactions, varied spacing, and strong typographic hierarchy — never with neon shadows or gradient text.
2. **Coral is the signal.** `#FF4D6D` marks the one primary action or key datum per view; everything else stays quiet. No competing accents, no every-button-primary.
3. **Trust is earned in the details.** Stripe-grade polish: aligned optical spacing, real empty states that teach, precise UX copy, no redundant headers or filler decoration.
4. **Show, don't gate.** Progressive disclosure — the 60-second happy path stays dead simple; sophistication (channels, agency tooling) reveals itself on demand.
5. **WCAG AA is the floor.** 4.5:1 text contrast on all surfaces (verify coral-on-navy combinations), visible focus rings, full keyboard navigation, `prefers-reduced-motion` respected on all animation.

### Technical Notes

- Stack: Next.js App Router, plain JS, Tailwind CSS + shadcn/ui (Radix), `tailwindcss-animate`
- Tokens live in `app/globals.css` as raw hex CSS variables; tailwind.config.js expects shadcn-style `hsl(var(--*))` tokens that are not currently defined — reconcile before leaning on shadcn semantic color classes
- Fonts loaded via `next/font` with `--font-display` / `--font-body` variables
